05 — Workqueue Driver

Deferred execution · workqueueschedule_workprocess contexttimer + work

Overview

Separates the timer ISR from the actual work. The timer callback fires in softirq context — it does nothing except schedule a work item. The workqueue executes in process context, where it can safely write to the ring buffer and call wake_up. This is the standard pattern for deferring non-trivial work out of interrupt context.

Two-Stage Design

Stage 1 — Timer callback (softirq, minimal)

void timer_callback(struct timer_list *data)
{
    schedule_work(&workqueue);   /* hand off to process context */

    if (kernel_logger.timer_active)
        mod_timer(&etx_timer, jiffies + msecs_to_jiffies(TIMEOUT));
}

Stage 2 — Work function (process context, does real work)

void workqueue_fn(struct work_struct *work)
{
    unsigned long flags;

    spin_lock_irqsave(&kernel_logger.lock, flags);
    snprintf(kernel_logger.kernel_buffer[kernel_logger.write_indexer],
             MEM_SIZE, "worker_event_%ld\n", kernel_logger.timer_count++);
    kernel_logger.write_indexer =
        (kernel_logger.write_indexer + 1) % ROW_SIZE;
    if (kernel_logger.count < ROW_SIZE)
        kernel_logger.count++;
    else
        kernel_logger.read_indexer =
            (kernel_logger.read_indexer + 1) % ROW_SIZE;
    spin_unlock_irqrestore(&kernel_logger.lock, flags);

    wake_up_interruptible(&etx_wait_queue);
}

Static Work Declaration

DECLARE_WORK(workqueue, workqueue_fn);
Static declaration is fine for a single, reusable work item. Dynamic allocation with INIT_WORK is used when work items need to be created per-request.

Clean Exit

On module unload, the timer is stopped first, then any pending work is flushed — guaranteeing the workqueue function completes before resources are freed.

static void __exit etx_driver_exit(void)
{
    del_timer_sync(&etx_timer);   /* stop timer, wait for callback */
    flush_work(&workqueue);       /* wait for any queued work item */
    device_destroy(dev_class, dev);
    /* ... */
}
