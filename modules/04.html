04 — Kernel Timer Driver

Async event generation · timer_listsoftirq contextspinlock_irqsavedel_timer_sync

Overview

A kernel timer fires every 1000ms and writes timer_event_N into a ring buffer. Userspace reads events via blocking read(). Timer start/stop is controlled via ioctl(). This introduces the critical constraint: timer callbacks run in softirq context — mutex is forbidden, spinlock is mandatory.

ParameterValue
Device node/dev/sanath_timer
Timer interval1000 ms
Buffer16 rows × 128 bytes
Synchronizationspinlock_t with irqsave/irqrestore

IOCTL magic'f'

Spinlock vs Mutex — Context Matters

The timer callback runs in softirq context. Mutexes can sleep — illegal in softirq. Spinlocks are the only option.

PrimitiveCan SleepSafe in Softirq
mutexYes❌ No
spinlock_tNo✅ Yes
unsigned long flags;
spin_lock_irqsave(&kernel_logger.lock, flags);
/* critical section — no sleeping, no copy_to_user */
spin_unlock_irqrestore(&kernel_logger.lock, flags);
irqsave/irqrestore saves the CPU interrupt enable state before locking and restores it exactly on unlock — required when the lock is shared between process context and interrupt context.

What Cannot Run Under Spinlock

FunctionReasonSafe?
wake_up_interruptibleTouches scheduler❌ Move outside
copy_to_userMay page fault (sleep)❌ Never
kmalloc(GFP_KERNEL)May sleep❌ Never
mod_timerHas internal locks⚠️ Move outside
snprintf / memcpyNo side effects✅ Yes

Correct copy_to_user pattern

uint8_t tmp[MEM_SIZE];

spin_lock_irqsave(&lock, flags);
memcpy(tmp, buffer[idx], copy_len);   /* copy inside lock */
memset(buffer[idx], 0, MEM_SIZE);
count--;
spin_unlock_irqrestore(&lock, flags);

if (copy_to_user(user_buf, tmp, copy_len))  /* outside lock */
    return -EFAULT;

Timer Active Flag

del_timer_sync() waits for any in-progress callback to finish, but it cannot prevent a callback that has already started from re-arming the timer. The timer_active flag solves this.

/* STOP ioctl */
kernel_logger.timer_active = 0;   /* clear first */
del_timer_sync(&etx_timer);       /* then wait */

/* callback — only re-arm if still active */
if (kernel_logger.timer_active)
    is_active = 1;
spin_unlock_irqrestore(&lock, flags);

if (is_active)
    mod_timer(&etx_timer, jiffies + msecs_to_jiffies(TIMEOUT));
Using del_timer instead of del_timer_sync is a kernel crash waiting to happen — the callback may still be executing on another CPU when the module unloads.

Bugs Encountered and Fixed
