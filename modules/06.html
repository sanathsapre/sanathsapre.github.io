06 — Workqueue + Poll Driver

Complete async pipeline · workqueuepollO_NONBLOCKspinlock

Overview

Combines all mechanisms from drivers 02–05 into a single coherent pipeline: a kernel timer fires → schedules a workqueue item → work writes to ring buffer → wakes up blocked/polling readers. Userspace can access data via blocking read, non-blocking read, or poll()/select(). This architecture directly models a camera frame pipeline.

Poll with Spinlock

Unlike driver 02 which used a mutex-based approach, here the poll function must use a spinlock because the workqueue also holds a spinlock when updating count.

static __poll_t etx_poll(struct file *filp, poll_table *wait)
{
    unsigned long flags;
    __poll_t mask = 0;

    poll_wait(filp, &etx_wait_queue, wait);

    spin_lock_irqsave(&kernel_logger.lock, flags);
    if (kernel_logger.count > 0)
        mask = POLLIN | POLLRDNORM;
    spin_unlock_irqrestore(&kernel_logger.lock, flags);

    return mask;
}
The spinlock inside poll() is necessary here — count is updated from workqueue context which also holds the spinlock. Consistency requires the same lock everywhere.

Non-Blocking Read

while (kernel_logger.count == 0) {
    spin_unlock_irqrestore(&kernel_logger.lock, flags);

    if (filp->f_flags & O_NONBLOCK)
        return -EAGAIN;

    if (wait_event_interruptible_exclusive(etx_wait_queue,
                                           kernel_logger.count > 0))
        return -ERESTARTSYS;

    spin_lock_irqsave(&kernel_logger.lock, flags);
}

Full Data Flow

timer_callback()          /* softirq — fires every 1000ms */
  └─ schedule_work()
       └─ workqueue_fn()  /* process context */
            ├─ spin_lock_irqsave
            ├─ snprintf → ring buffer
            ├─ advance write_indexer, update count
            ├─ spin_unlock_irqrestore
            └─ wake_up_interruptible → unblocks read() or poll()
