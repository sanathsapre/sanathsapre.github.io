Linux Kernel
Driver Lab

Seven progressively complex kernel modules built to understand the primitives that underpin real camera pipelines — from basic character drivers to real hardware GPIO on BeagleBone Black.

Module 01
Message Queue Driver
Kernel-user FIFO communication via a 16-slot ring buffer
character driver ring buffer wait queue mutex
Module 02
Non-Blocking + Poll Driver
Event-driven I/O — O_NONBLOCK, poll(), select() on the same device node
O_NONBLOCK poll/select wait queue EAGAIN
Module 03
IOCTL Interface
Out-of-band control plane — the same pattern used by V4L2, ALSA, and USB drivers
unlocked_ioctl _IO/_IOR/_IOW magic number
Module 04
Kernel Timer Driver
Async event generation — introduces softirq context and mandatory spinlock use
timer_list softirq context spinlock_irqsave del_timer_sync
Module 05
Workqueue Driver
Deferred execution — timer schedules work items that run safely in process context
workqueue schedule_work process context DECLARE_WORK
Module 06
Workqueue + Poll Driver
Complete async pipeline — timer → workqueue → ring buffer → poll/read. Camera frame model.
workqueue poll O_NONBLOCK spinlock
Module 07
BBB GPIO Driver
Real hardware — controls the USR0 LED on BeagleBone Black via GPIO1_21
gpio_request AM335x USR LED gpio_export
