Overview

Extends the message queue driver with non-blocking read support and poll()/select() integration. Userspace can now choose between blocking, non-blocking, and event-driven access patterns on the same device node.

Non-Blocking Read

When the caller opens with O_NONBLOCK, an empty buffer returns immediately with -EAGAIN instead of sleeping. This is the standard POSIX contract for non-blocking I/O.

while (kernel_logger.count == 0) {
    mutex_unlock(&kernel_logger.etx_mutex);
    if (filp->f_flags & O_NONBLOCK)
        return -EAGAIN;
    if (wait_event_interruptible_exclusive(etx_wait_queue,
                                           kernel_logger.count > 0))
        return -ERESTARTSYS;
    mutex_lock(&kernel_logger.etx_mutex);
}

Poll Implementation

poll_wait() registers the wait queue with the kernel's polling infrastructure. The driver then checks count and returns the appropriate mask — no sleeping inside poll().

static __poll_t etx_poll(struct file *filp, poll_table *wait)
{
    etx_priv_t *priv = filp->private_data;
    poll_wait(filp, &etx_wait_queue, wait);
    if (kernel_logger.count > 0)
        return POLLIN | POLLRDNORM;
    return 0;
}
poll_wait() does NOT sleep. It only registers the wait queue so the kernel knows to wake this fd when data arrives.
