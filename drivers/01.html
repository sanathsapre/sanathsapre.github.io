Message Queue Driver

Kernel-user FIFO communication

Overview

A character driver that exposes a 16-slot × 128-byte FIFO ring buffer to userspace via /dev/sanath_queue. Writers push messages; readers block until data is available. Models the producer-consumer pattern at the core of camera pipeline data paths.

Ring Buffer Design

Two independent index pointers — read_indexer and write_indexer — advance modulo ROW_SIZE. A count field tracks occupancy explicitly, avoiding the index-equality ambiguity between empty and full states.

typedef struct kernel_logger {
    uint8_t kernel_buffer[ROW_SIZE][MEM_SIZE];
    uint8_t read_indexer;
    uint8_t write_indexer;
    struct mutex etx_mutex;
    uint8_t count;
} kernel_logger_t;

Drop-Oldest on Full

if (kernel_logger.count < ROW_SIZE)
    kernel_logger.count++;
else
    kernel_logger.read_indexer =
        (kernel_logger.read_indexer + 1) % ROW_SIZE;
This models real-time streaming systems where freshness matters more than completeness — identical to camera frame drop policy when the consumer is slow.

Blocking Read

The reader sleeps on an empty queue and wakes only when a writer adds data. wait_event_interruptible_exclusive ensures only one reader wakes per event, avoiding thundering herd.

while (kernel_logger.count == 0) {
    mutex_unlock(&kernel_logger.etx_mutex);
    if (wait_event_interruptible_exclusive(etx_wait_queue,
                                           kernel_logger.count > 0))
        return -ERESTARTSYS;
    mutex_lock(&kernel_logger.etx_mutex);
}

Per-Open Session State

Each open() allocates a done flag via kmalloc and stores it in file->private_data. This ensures a reader gets exactly one message per open.

static int etx_open(struct inode *inode, struct file *file)
{
    int *done = kmalloc(sizeof(int), GFP_KERNEL);
    if (!done) return -ENOMEM;
    *done = 0;
    file->private_data = done;
    return 0;
}
