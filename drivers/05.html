MODULE 05

Workqueue Driver

Deferred execution

workqueueschedule_workprocess contexttimer + work

Overview

Separates the timer ISR from actual work. The timer callback fires in softirq context and does nothing except schedule a work item. The workqueue executes in process context where it can safely write to the ring buffer and call wake_up. Standard pattern for deferring non-trivial work out of interrupt context.

Two-Stage Design

Stage 1 — Timer callback (softirq, minimal)

void timer_callback(struct timer_list *data)
{
    schedule_work(&workqueue);
    if (kernel_logger.timer_active)
        mod_timer(&etx_timer, jiffies + msecs_to_jiffies(TIMEOUT));
}

Stage 2 — Work function (process context)

void workqueue_fn(struct work_struct *work)
{
    unsigned long flags;
    spin_lock_irqsave(&kernel_logger.lock, flags);
    snprintf(kernel_logger.kernel_buffer[kernel_logger.write_indexer],
             MEM_SIZE, "worker_event_%ld
", kernel_logger.timer_count++);
    kernel_logger.write_indexer = (kernel_logger.write_indexer + 1) % ROW_SIZE;
    if (kernel_logger.count < ROW_SIZE) kernel_logger.count++;
    else kernel_logger.read_indexer = (kernel_logger.read_indexer + 1) % ROW_SIZE;
    spin_unlock_irqrestore(&kernel_logger.lock, flags);
    wake_up_interruptible(&etx_wait_queue);
}

Clean Exit

On module unload, the timer is stopped first, then any pending work is flushed — guaranteeing the workqueue function completes before resources are freed.

del_timer_sync(&etx_timer);   /* stop timer */
flush_work(&workqueue);       /* wait for queued work */
