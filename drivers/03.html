MODULE 03

IOCTL Interface

Kernel control plane

unlocked_ioctl_IO/_IOR/_IOWmagic number

Overview

Adds an ioctl() control interface on top of the message queue driver. Userspace can query queue state, clear it, and reset the driver — without using read/write. This is the standard Linux pattern for out-of-band control commands (used heavily in V4L2, ALSA, and USB drivers).

ParameterValue
Device node/dev/sanath_queue
IOCTL magic'e'
InheritsNon-blocking + poll from driver 02

IOCTL Commands

CommandMacroDirectionDescription
GET_QUEUE_SIZE_IOR('e',1,int32_t)kernel → userCurrent number of messages
GET_MAX_CAPACITY_IOR('e',2,int32_t)kernel → userMax queue depth (16)
CLEAR_QUEUE_IO('e',3)noneReset indices and count
RESET_DEVICE_IO('e',4)noneFull driver state reset

Macro Encoding

The kernel encodes direction, magic, sequence number, and data size into a single 32-bit command number, preventing cross-driver collisions.

#define ETX_MAGIC        'e'
#define GET_QUEUE_SIZE   _IOR(ETX_MAGIC, 1, int32_t)
#define GET_MAX_CAPACITY _IOR(ETX_MAGIC, 2, int32_t)
#define CLEAR_QUEUE      _IO(ETX_MAGIC,  3)
#define RESET_DEVICE     _IO(ETX_MAGIC,  4)
MacroDirection
_IONo data
_IORDriver → userspace
_IOWUserspace → driver
_IOWRBidirectional

Handler Implementation

static long etx_ioctl(struct file *file, unsigned int cmd, unsigned long arg)
{
    int32_t val = 0;
    switch (cmd) {
    case GET_QUEUE_SIZE:
        mutex_lock(&kernel_logger.etx_mutex);
        val = kernel_logger.count;
        mutex_unlock(&kernel_logger.etx_mutex);
        if (copy_to_user((int32_t __user *)arg, &val, sizeof(val)))
            return -EFAULT;
        break;
    case CLEAR_QUEUE:
        mutex_lock(&kernel_logger.etx_mutex);
        memset(kernel_logger.kernel_buffer, 0, sizeof(kernel_logger.kernel_buffer));
        kernel_logger.read_indexer = kernel_logger.write_indexer = kernel_logger.count = 0;
        mutex_unlock(&kernel_logger.etx_mutex);
        break;
    default: return -EINVAL;
    }
    return 0;
}
